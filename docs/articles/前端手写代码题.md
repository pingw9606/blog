## 一、防抖 Debounce（最高频）

**场景**：搜索输入、resize、按钮防连点。连续触发只在「停止 N 毫秒后」执行一次。

```js
function debounce(fn, delay = 300) {
  let timer = null;
  return function (...args) {
    clearTimeout(timer);
    timer = setTimeout(() => fn.apply(this, args), delay);
  };
}
```
**进阶（带立即执行 + 取消）**：
```js
function debounce(fn, delay = 300, immediate = false) {
  let timer = null;
  function debounced(...args) {
    if (timer) clearTimeout(timer);
    if (immediate && !timer) fn.apply(this, args);
    timer = setTimeout(() => {
      timer = null;
      if (!immediate) fn.apply(this, args);
    }, delay);
  }
  debounced.cancel = () => { clearTimeout(timer); timer = null; };
  return debounced;
}
```
> 考点：闭包保存 timer、`this`/`args` 透传、清除旧定时器。

## 二、节流 Throttle

**场景**：滚动监听、高频拖拽。固定时间间隔最多执行一次。

```js
// 时间戳版（首次立即执行）
function throttle(fn, interval = 300) {
  let last = 0;
  return function (...args) {
    const now = Date.now();
    if (now - last >= interval) {
      last = now;
      fn.apply(this, args);
    }
  };
}
```
> 面试常追问「防抖和节流区别」：防抖=「等你停下来再做」（合并），节流=「固定频率做」（稀释）。

## 三、useDebounce / useDebouncedCallback（React 版，本项目同款思路）

```tsx
// 防抖值
function useDebounce<T>(value: T, delay = 300): T {
  const [debounced, setDebounced] = useState(value);
  useEffect(() => {
    const t = setTimeout(() => setDebounced(value), delay);
    return () => clearTimeout(t);   // value 变/卸载时清掉上一个
  }, [value, delay]);
  return debounced;
}

// 用法：搜索
const debouncedQuery = useDebounce(query, 250);
useEffect(() => { if (debouncedQuery) fetchResults(debouncedQuery); }, [debouncedQuery]);
```
> 考点：useEffect 清理函数 + 依赖。本项目 SearchBox 是直接在 effect 里 setTimeout（等价写法）。

## 四、带请求取消的 useFetch（竞态处理，高质量题）

```tsx
function useFetch<T>(url: string) {
  const [data, setData] = useState<T | null>(null);
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState<Error | null>(null);

  useEffect(() => {
    const ctrl = new AbortController();
    setLoading(true);
    fetch(url, { signal: ctrl.signal })
      .then(r => r.json())
      .then(d => setData(d))
      .catch(e => { if (e.name !== 'AbortError') setError(e); })
      .finally(() => setLoading(false));
    return () => ctrl.abort();   // url 变/卸载，取消上一个请求，避免旧结果覆盖新结果
  }, [url]);

  return { data, loading, error };
}
```
> 考点：**竞态条件**（race condition）——快速切换 url 时旧请求可能后返回。用 AbortController 取消，或用 `ignore` 标志位。这是面试区分新手/老手的题。

## 五、实现简化版 useState（理解 hooks 原理）

```js
let states = [];
let cursor = 0;

function useState(initial) {
  const i = cursor;                       // 闭包记住本 hook 的位置
  states[i] = states[i] ?? initial;
  const setState = (val) => {
    states[i] = typeof val === 'function' ? val(states[i]) : val;
    cursor = 0;                           // 重置游标
    render();                             // 触发重渲染
  };
  cursor++;
  return [states[i], setState];
}
```
> 考点：**为什么 hooks 不能写在条件里**——靠 `cursor`（调用顺序）关联状态，顺序变了就错位。这题答出来直接体现你懂原理。

## 六、useInterval（闭包陷阱经典题，Dan Abramov 名题）

直接 `setInterval` + setState 会有闭包陷阱（回调里的 state 永远是初始值）。正确做法用 ref 存最新回调：
```tsx
function useInterval(callback: () => void, delay: number | null) {
  const savedRef = useRef(callback);
  useEffect(() => { savedRef.current = callback; }, [callback]); // 每次渲染更新最新回调
  useEffect(() => {
    if (delay === null) return;
    const id = setInterval(() => savedRef.current(), delay);
    return () => clearInterval(id);
  }, [delay]);
}
```
> 考点：闭包陷阱 + useRef 存「可变最新值」+ 清理定时器。

## 七、Promise 并发控制（限制最大并发数，高频进阶）

**场景**：100 个请求，最多同时 3 个。
```js
async function asyncPool(limit, tasks) {
  const results = [];
  const executing = new Set();
  for (const [i, task] of tasks.entries()) {
    const p = Promise.resolve().then(() => task());
    results[i] = p;
    executing.add(p);
    p.finally(() => executing.delete(p));
    if (executing.size >= limit) {
      await Promise.race(executing);   // 等任意一个完成再继续
    }
  }
  return Promise.all(results);
}
// 用法：asyncPool(3, urls.map(u => () => fetch(u)));
```
> 考点：`Promise.race` 控制并发窗口、Set 管理在途任务。

## 八、手写 Promise.all / allSettled

```js
function myAll(promises) {
  return new Promise((resolve, reject) => {
    const res = [];
    let count = 0;
    if (promises.length === 0) return resolve(res);
    promises.forEach((p, i) => {
      Promise.resolve(p).then(
        v => { res[i] = v; if (++count === promises.length) resolve(res); },
        reject                                  // 任一失败立即 reject
      );
    });
  });
}

function myAllSettled(promises) {
  return Promise.all(
    promises.map(p =>
      Promise.resolve(p).then(
        value => ({ status: 'fulfilled', value }),
        reason => ({ status: 'rejected', reason })  // 不 reject，包装结果
      )
    )
  );
}
```
> 考点：保持结果**顺序**（用 index 不用 push）、计数判完成、all vs allSettled 的失败处理差异。

## 九、事件循环输出题（必考）

```js
console.log(1);
setTimeout(() => console.log(2), 0);          // 宏任务
Promise.resolve().then(() => console.log(3)); // 微任务
console.log(4);
// 输出：1 4 3 2
```
**规则**：同步代码 → 清空所有微任务（Promise.then/queueMicrotask）→ 取一个宏任务（setTimeout/setInterval/IO）→ 再清空微任务 …循环。
> 进阶题常混入 async/await（`await` 后面的代码等价于 `.then` 里的微任务）。

```js
async function f() {
  console.log('a');
  await Promise.resolve();
  console.log('b');     // 微任务
}
console.log('start');
f();
console.log('end');
// 输出：start → a → end → b
```

## 十、React 重渲染输出题（判断会不会重渲染）

```tsx
function Parent() {
  const [count, setCount] = useState(0);
  const handleClick = () => {};               // ❌ 每次渲染新引用
  return <>
    <button onClick={() => setCount(count+1)}>+</button>
    <Child onClick={handleClick} />            {/* 即使 memo 也会重渲染 */}
  </>;
}
const Child = React.memo(({ onClick }) => { console.log('render'); return <div/>; });
```
**问**：点按钮，Child 会 render 吗？**会**——因为 `handleClick` 每次是新函数，memo 浅比较失败。
**修复**：`const handleClick = useCallback(() => {}, [])`。
> 考点：memo 失效原因 + useCallback。

## 十一、深拷贝（含循环引用）

```js
function deepClone(obj, map = new WeakMap()) {
  if (obj === null || typeof obj !== 'object') return obj;
  if (obj instanceof Date) return new Date(obj);
  if (obj instanceof RegExp) return new RegExp(obj);
  if (map.has(obj)) return map.get(obj);        // 循环引用
  const clone = Array.isArray(obj) ? [] : {};
  map.set(obj, clone);
  for (const key of Reflect.ownKeys(obj)) {
    clone[key] = deepClone(obj[key], map);
  }
  return clone;
}
```
> 考点：循环引用用 WeakMap、特殊对象处理、Symbol key（Reflect.ownKeys）。

## 十二、虚拟列表（思路题，能讲清即可）

**问题**：1 万条数据全渲染会卡。
**思路**：
1. 容器固定高度、可滚动；只渲染**可视区 + 缓冲区**的若干条。
2. 监听 scroll，根据 `scrollTop / itemHeight` 算出起始 index。
3. 用一个撑高的占位元素（`height = total * itemHeight`）保持滚动条正确。
4. 可视项用 `transform: translateY(startIndex * itemHeight)` 定位。
```
startIndex = Math.floor(scrollTop / itemHeight)
endIndex   = startIndex + Math.ceil(viewportHeight / itemHeight) + buffer
渲染 list.slice(startIndex, endIndex)，整体 translateY(startIndex*itemHeight)
```
> 不定高用「预估高度 + 测量缓存」。生产用 `react-window`/`react-virtualized`。

## 十三、实现 useLocalStorage（自定义 hook）

```tsx
function useLocalStorage<T>(key: string, initial: T) {
  const [value, setValue] = useState<T>(() => {
    if (typeof window === 'undefined') return initial;   // SSR 安全
    const raw = localStorage.getItem(key);
    return raw ? JSON.parse(raw) : initial;
  });
  useEffect(() => {
    localStorage.setItem(key, JSON.stringify(value));
  }, [key, value]);
  return [value, setValue] as const;
}
```
> 考点：useState 惰性初始化（传函数，只算一次）、SSR 下 `window` 判空、序列化。

## 十四、柯里化 / 偏函数（基础但常考）

```js
function curry(fn) {
  return function curried(...args) {
    if (args.length >= fn.length) return fn.apply(this, args);
    return (...rest) => curried.apply(this, [...args, ...rest]);
  };
}
const add = curry((a, b, c) => a + b + c);
add(1)(2)(3); add(1, 2)(3); add(1)(2, 3);   // 都 = 6
```

## 十五、手写 instanceof / new / call/apply/bind（JS 基础）

```js
function myInstanceof(obj, Ctor) {
  let proto = Object.getPrototypeOf(obj);
  while (proto) {
    if (proto === Ctor.prototype) return true;
    proto = Object.getPrototypeOf(proto);
  }
  return false;
}

Function.prototype.myBind = function (ctx, ...preset) {
  const fn = this;
  return function (...args) {
    return fn.apply(ctx, [...preset, ...args]);
  };
};
```
