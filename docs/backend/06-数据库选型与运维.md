# 06 · 数据库：选型与运维

## 一、选型

| 选项 | 适用 | 说明 |
|------|------|------|
| **PostgreSQL** | 中大型/结构化数据 | 功能全、全文检索强，本项目所用 |
| **MySQL** | 同上，更普及 | 你们公司所用，理念相通 |
| SQLite | 极小型/纯静态 | 一个文件，多用户写入会成瓶颈 |
| MongoDB | 文档型 | 强关系数据反而不顺手 |

**小说站这类"一本书有很多章、一个用户发很多评论"= 典型关系型**，PostgreSQL / MySQL 都合适。

## 二、ORM 而不是手写 SQL
后端应用通过 **ORM** 查库，不手拼 SQL 字符串：

| 本项目 | Java |
|--------|------|
| Prisma | JPA/Hibernate 或 MyBatis |
| `prisma.user.findFirst({where:{email}})` | `userRepository.findByEmail(email)` |

ORM 好处：**类型安全**、**参数化查询防 SQL 注入**、跨数据库可移植。
ORM 底层仍发参数化 SQL 给数据库。逃生口：`prisma.$queryRaw` / MyBatis 原生 SQL。

## 三、迁移（管理表结构）
**改表结构不靠手敲 `ALTER TABLE`，靠版本化迁移脚本**：

| 本项目 | Java |
|--------|------|
| Prisma Migrate（`prisma/migrations/`） | Flyway / Liquibase |
| `npm run db:migrate`（开发） | `flyway migrate` |
| `npm run db:deploy`（生产） | CI 中执行迁移 |

好处：表结构变更有版本记录、可重放、团队同步。

## 四、连接池
应用启动时建一组数据库连接反复复用，不是每次查询新建连接：

| 本项目 | Java |
|--------|------|
| PrismaClient 单例（`server/utils/prisma.ts`） | HikariCP |

`psql`/`mysql` 命令行直连是临时单连接，查完即断——只用于调试。

## 五、迁移到别的库/服务器麻不麻烦
1. **同种库换服务器/备份** → 简单：`pg_dump`/`mysqldump` 导出再导入；Docker 直接搬数据卷。
2. **换引擎（如 SQLite→PostgreSQL）** → 中等：数据有工具（pgloader），代码层若用 ORM 改动很小，裸 SQL 改动大。
3. **降低成本的做法**：用 ORM + 迁移工具 + 数据放 Docker 卷 + 不依赖某库专有特性。

## 六、数据库安全铁律
- **端口绝不裸露公网**：只绑 `127.0.0.1` 或走内网，ufw/安全组都别开 5432/3306。
- 密码用强随机值，不入 Git（放 `.env`，`.env` 进 `.gitignore`）。
- 生产改数据走工单/审批，不直接手敲 `UPDATE/DELETE`。
