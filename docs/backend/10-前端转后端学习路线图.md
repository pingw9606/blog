# 10 · 前端转后端学习路线图（Java 方向）

> 给有前端基础、目标是 Java + MySQL 后端的同学。按阶段循序渐进。

## 阶段 0 · 心态与优势
你已有的可迁移能力：HTTP、JSON、API 调用、Git、调试思维、工程化。
要补的核心差异：**多线程并发、数据库、JVM、服务端架构**。

## 阶段 1 · Java 语言基础（2-4 周）
- 语法：类/接口/泛型/异常/集合（List/Map/Set）
- **面向对象**：封装/继承/多态（前端较弱的部分，重点补）
- Java 8 特性：Lambda、Stream、Optional（和 JS 数组方法类似，易上手）
- 工具：JDK、Maven/Gradle（≈ npm）、IntelliJ IDEA

## 阶段 2 · Spring Boot 全家桶（4-6 周，重中之重）
| 知识 | 对应前面笔记 | 类比 |
|------|------------|------|
| Spring Boot 起步、注解 | — | 约定大于配置 |
| Controller（REST API） | [05 分层] | Nuxt 的 server/api |
| Service / 依赖注入(IoC) | [05] | 业务层 |
| **MyBatis / JPA** 访问数据库 | [06 ORM] | Prisma |
| 事务 @Transactional | [import 用的事务] | 一组操作同成同败 |
| Spring Security（鉴权） | [05 鉴权原理] | nuxt-auth-utils |
| 参数校验、统一异常处理 | — | — |

做一个完整 CRUD 项目（如你这个小说站的 Java 版）。

## 阶段 3 · 数据库 & SQL（持续）
- SQL 熟练：增删改查、JOIN、聚合、子查询
- **索引优化** → 见 [09]
- 事务、隔离级别、锁
- **查库找问题流程** → 见 [07]
- 连接池（HikariCP）、读写分离概念

## 阶段 4 · 并发与 JVM（进阶）
- 线程、线程池、`synchronized`/`Lock`、`volatile`、CAS
- JUC 常用类：`ConcurrentHashMap`、`CompletableFuture`
- **JVM 内存与 GC、线上排查** → 见 [08]
- 这部分是后端和前端差异最大、面试最爱问的

## 阶段 5 · 中间件与分布式（按需）
- **Redis**：缓存、分布式锁、排行榜（小说站的阅读量/热榜很合适）
- **消息队列**：Kafka/RabbitMQ（异步解耦）
- 微服务：Spring Cloud、服务注册、网关、配置中心
- 搜索：Elasticsearch

## 阶段 6 · 运维与上线（你已经在练了）
- Linux 基础 + 排查 → 见 [01][02]
- Git/协作 → 见 [03]
- **Docker + CI/CD** → 见 [04]
- 监控（Prometheus/Grafana）、日志（ELK）、链路追踪（SkyWalking）

## 推荐节奏
```
1-2 月：Java 基础 + Spring Boot，做出能跑的 CRUD 项目
3 月：  数据库/SQL/索引 + Redis，给项目加缓存
4 月：  并发 + JVM 基础，看懂线上排查
之后：  分布式/微服务按工作需要补
```

## 学习方法建议
1. **以项目驱动**：把这个小说站用 Java + Spring Boot + MySQL 重写一遍，是最好的练手——需求你已经完全懂了，只需关注"换成 Java 怎么实现"。
2. **对照学**：每学一个 Java 概念，回头看它在本笔记里对应的 Node/Prisma 实现，迁移最快。
3. **动手排查**：故意制造慢 SQL、内存泄漏、报错，练 EXPLAIN、jstack、看日志。
4. 面试前重点准备：集合源码、并发、JVM、MySQL 索引与事务、Spring 原理。

## 一句话
**你缺的不是"会不会写代码"，而是"服务端特有的那套：并发、数据库、JVM、分布式"。把这个小说站用 Java 重写一遍 + 啃透这几块，就完成了转型。**
