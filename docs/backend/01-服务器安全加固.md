# 01 · 服务器安全加固

一台新服务器（Ubuntu）拿到手，最先该做的安全加固。

## 名词速懂（前端类比）

| 名词 | 一句话 | 前端类比 |
|------|--------|---------|
| SSH | 远程登录服务器的加密通道 | git push 走的加密连接 |
| 密钥登录 | 公钥/私钥代替密码，更安全 | GitHub deploy key |
| `sudo` | 以管理员身份执行 | 全局安装要的管理员权限 |
| `ufw` | 防火墙，控制端口对外开放 | 只开放允许的接口 |
| `fail2ban` | 自动封禁反复猜密码的 IP | 登录接口限流 + 拉黑 |
| 端口 | 服务的"门牌号"，SSH 默认 22 | localhost:3000 的 3000 |

## 三步加固（顺序很重要）

### Step A — 开防火墙（先放行 SSH 再开，否则锁死自己）
```bash
sudo ufw default deny incoming      # 外部进来默认拒绝
sudo ufw default allow outgoing     # 出站允许
sudo ufw limit 22/tcp comment 'SSH' # 放行并限速 SSH（limit 比 allow 抗爆破）
sudo ufw --force enable             # --force 跳过交互确认
sudo ufw status verbose
```

### Step B — fail2ban 自动封禁
```bash
sudo DEBIAN_FRONTEND=noninteractive apt-get -y -o Dpkg::Options::=--force-confold install fail2ban
```
配置 `/etc/fail2ban/jail.local`：
```ini
[DEFAULT]
bantime  = 1h
findtime = 10m
maxretry = 5
backend  = systemd
[sshd]
enabled = true
port    = 22
```
```bash
sudo systemctl enable --now fail2ban
sudo fail2ban-client status sshd
```

### Step C — SSH 加固（先校验再生效）
```bash
sudo cp -a /etc/ssh/sshd_config /etc/ssh/sshd_config.bak.$(date +%F)
sudo tee /etc/ssh/sshd_config.d/99-hardening.conf >/dev/null <<'CFG'
PubkeyAuthentication yes
PasswordAuthentication no          # 关闭密码登录，只认密钥
KbdInteractiveAuthentication no
PermitRootLogin prohibit-password  # root 只能用密钥
MaxAuthTries 4
LoginGraceTime 30
X11Forwarding no
CFG
sudo sshd -t && sudo systemctl reload ssh   # 先校验语法，OK 才 reload
```

## 验证（关旧连接前必做）
```bash
# 新密钥登录应成功
ssh -i ~/.ssh/id_ed25519 ubuntu@<IP> 'echo OK'
# 密码登录应被拒
ssh -o PreferredAuthentications=password -o PubkeyAuthentication=no ubuntu@<IP>
# 期望：Permission denied (publickey)
```

## ⚠️ 关键提醒
1. **改 SSH/防火墙前，留一个能进去的兜底通道**（如云厂商网页终端），防止锁死。
2. **`reload` 不断开现有连接，`restart` 会**——加固用 reload 更稳。
3. **密钥丢了就进不去**，务必备份私钥。
4. **云服务器有两层防火墙**：本机 ufw + 云控制台安全组，开端口两处都要放行。
