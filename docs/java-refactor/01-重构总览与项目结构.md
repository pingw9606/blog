# 01 · 重构总览与项目结构

本篇把"从哪开始、怎么拆"讲清楚：先摸清旧项目，再规划新结构和仓库。

## 一、先读懂旧项目

旧的 Nuxt 全栈长这样（关键部分）：

```
ouyang-app/
├─ pages/                # Vue 页面（基于文件的路由）
│  ├─ index.vue          #   书架
│  ├─ novel/[slug].vue   #   小说详情
│  ├─ chapter/[id].vue   #   章节阅读
│  ├─ login.vue / register.vue / account.vue
│  ├─ author/ / admin/
├─ server/api/           # 后端接口（Nitro）
│  ├─ auth/              #   注册/登录/登出/当前用户
│  ├─ novels/ chapters/ comments/ volumes/
│  ├─ author-applications/ admin/ reading-progress/
│  └─ utils/             #   prisma 客户端、权限守卫 guards.ts
├─ prisma/schema.prisma  # 数据模型
└─ ...
```

业务实体和角色（这是重构的"锚"，前后端都不变）：

- **User**（角色 `reader` / `author` / `admin`，状态 `active` / `banned`）
- **Novel**（小说）— **Volume**（分卷）— **Chapter**（章节，Markdown 正文）
- **Comment**（评论）、**AuthorApplication**（作者申请）、**ReadingProgress**（阅读进度）

核心规则：
- 读者可申请成为作者 → 管理员审批通过后升级为 author。
- 小说/章节有 `draft` / `published`；公开只看已发布，作者本人/管理员可看自己的草稿。
- 登录用户可评论、记录阅读进度。

> 重构第一原则：**业务不变，只换技术栈**。把旧的接口和页面当"验收标准"，一个个搬到 Java/React 上。

## 二、规划新结构

前后端分离后，拆成两个独立项目：

```
ouyang-saga/                 # 本地工作区（放一起方便对照）
├─ legacy-nuxt/              # 旧项目（保留，独立 Git 仓库，参照用）
├─ ouyang-server/            # Java 后端（独立仓库）
└─ ouyang-web/               # React 前端（独立仓库）
```

后端按经典分层组织（下一篇细讲）：

```
ouyang-server/src/main/java/com/ouyang/saga/
├─ domain/       # JPA 实体（对应 Prisma model）+ enums
├─ repository/   # 数据访问（Spring Data JPA）
├─ dto/          # 请求/响应对象
├─ service/      # 业务逻辑（对应旧 server/api 的处理函数）
├─ controller/   # REST 控制器（对应旧路由文件）
├─ security/     # JWT + Spring Security
├─ common/       # 异常、工具、常量
└─ config/       # 种子数据等
```

前端按页面组织：

```
ouyang-web/src/
├─ api.ts        # axios 封装 + 各接口方法
├─ auth.tsx      # 登录态 Context
├─ types.ts      # 与后端 DTO 对应的类型
├─ App.tsx       # 路由 + 布局
└─ pages/        # 各页面（对应旧 pages/）
```

## 三、仓库拆分

旧项目本来在 GitHub 上叫 `ouyang-saga`。重构后的仓库规划：

1. 旧仓库改名归档：`ouyang-saga` → `ouyang-saga-nuxt-legacy`

```bash
gh repo rename ouyang-saga-nuxt-legacy -R pingw9606/ouyang-saga --yes
# 本地旧项目的 remote 也要跟着改
cd legacy-nuxt
git remote set-url origin git@github.com:pingw9606/ouyang-saga-nuxt-legacy.git
```

2. 新建两个独立仓库并推送：

```bash
# 后端
cd ouyang-server && git init -q && git branch -M main
git add . && git commit -q -m "初始化 Java 后端"
gh repo create pingw9606/ouyang-server --private --source=. --remote=origin --push

# 前端
cd ../ouyang-web && git init -q && git branch -M main
git add . && git commit -q -m "初始化 React 前端"
gh repo create pingw9606/ouyang-web --private --source=. --remote=origin --push
```

> **坑**：一开始图省事把 server+web 放一个"单体仓库"，后来发现两个独立项目各自独立发版、独立 CI 更清爽，于是拆成两个仓库、删掉单体的那个。**一开始就按前后端分离拆仓库，少走回头路。**
