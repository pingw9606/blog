# 转型 AI Agent 全栈工程师 · 自学教程大纲

> 面向有前端 / Node.js 基础的开发者，目标：用 TypeScript 全栈打通 Agent 开发，最终独立交付「企业级知识库」类 Agent 项目。
> 技术主线：LangChain / LangGraph / Vercel AI SDK（Agent 框架）+ Nest（后端）+ RAG / 记忆 / 向量库 / 知识图谱 / 可观测。

## 0. 学前准备
- **语言**：TypeScript / Node.js（≥ 20）、基础 HTTP/SSE 概念
- **环境**：pnpm、Docker Desktop、一个可用的大模型 API（OpenAI / DeepSeek / 通义等，OpenAI 兼容即可）
- **心态**：先跑通最小闭环，再逐步加工程化；每节都「能运行 + 有产出」
- **学习方法**：每节按 `目标 → 要点 → 动手产出` 推进，代码全部自己敲一遍

# 阶段一：Agent 开发基础（第 1–17 节）
> 用 LangChain 把「大模型 + 工具 + 知识 + 记忆」这套基本盘吃透。

### 1. AI Agent 开发要学什么？
- **目标**：建立全局地图，知道 Agent = LLM + Tool + Memory + 编排 + 知识。
- **要点**：Agent 与「纯聊天」的区别（能感知、能调用工具、能多步决策）；本教程的技术栈与最终项目预览。
- **产出**：一张自己的学习路线图 + 跑通第一个 LLM 调用（chat completion）。

### 2. 从 Tool 开始：让大模型自动调工具读文件
- **目标**：理解 function/tool calling 机制。
- **要点**：tool 的 schema（名称/描述/参数）；模型如何「决定」调用哪个工具并回填结果。
- **产出**：定义一个 `readFile` 工具，让模型自动读取本地文件并总结。

### 3. 实现 mini cursor：大模型自动调用 tool 执行命令
- **目标**：用「工具循环」实现一个会执行命令的迷你 agent。
- **要点**：tool-calling loop（模型→调用工具→把结果喂回→再决策），安全边界（命令白名单/确认）。
- **产出**：一个能根据自然语言执行 shell 命令的 mini cursor（带确认）。

### 4. MCP：可跨进程调用的 Tool
- **目标**：理解 MCP（Model Context Protocol）解决「工具标准化、可复用」的问题。
- **要点**：MCP Server / Client 架构；stdio / SSE 传输；与本地 tool 的区别。
- **产出**：跑通一个 MCP Server，并在客户端调用它的工具。

### 5. 高德 MCP + 浏览器 MCP：复用别人的 MCP Server
- **目标**：体会 MCP 生态的「即插即用」。
- **要点**：接入第三方 MCP（地图/浏览器），在 LangChain 中注册为可用工具。
- **产出**：一个能查路线 / 抓网页的 agent。

### 6. RAG：把文档向量化，基于向量实现语义搜索
- **目标**：搞懂 RAG 的本质——检索增强生成。
- **要点**：embedding、向量相似度、检索 → 拼 prompt → 生成；解决幻觉与私有知识问题。
- **产出**：把一份文档向量化，实现「按语义」而非关键词的搜索问答。

### 7. 知识库的 loader 和 splitter：加载并分割文档
- **目标**：掌握数据进入 RAG 前的预处理。
- **要点**：多来源 loader（md/pdf/网页）；为什么要 chunk；chunk size / overlap 的权衡。
- **产出**：一个支持多格式的文档导入管线。

### 8. LangChain 全部 Splitter，其实只需要其中一个
- **目标**：理清各类 splitter 的取舍。
- **要点**：递归字符切分为何是首选；按代码 / markdown 结构切分的场景。
- **产出**：对比不同 splitter 的检索效果，固化一套默认策略。

### 9. 向量数据库 Milvus：Agent 开发必备
- **目标**：从「内存向量」升级到生产级向量库。
- **要点**：Milvus 集合/索引（HNSW/IVF）、metric（cosine/L2）、为什么需要专用向量库。
- **产出**：用 Docker 起 Milvus，把文档向量写入并检索。

### 10. Milvus + RAG 实战：电子书语义检索助手
- **目标**：完成第一个完整 RAG 应用。
- **要点**：导入 → 切分 → 向量化 → 入库 → 检索 → 生成，端到端串起来。
- **产出**：一个可问答的电子书助手。

### 11. Memory 管理的三大策略：截断、总结、检索
- **目标**：让对话「记得住」。
- **要点**：上下文窗口限制；截断 / 滚动总结 / 向量检索三种记忆方案的适用场景。
- **产出**：给聊天 agent 加上可切换的记忆策略。

### 12. 结构化大模型输出：output parser 还是 tool？
- **目标**：拿到「可被程序消费」的结构化结果。
- **要点**：JSON schema 约束、output parser vs tool calling 两条路线的取舍。
- **产出**：让模型稳定输出结构化 JSON。

### 13. Output Parser 实战：智能录入 + 流式版 mini cursor
- **目标**：结构化输出落到真实场景。
- **要点**：表单智能录入；流式输出（streaming）与结构化的结合。
- **产出**：一个把自然语言转结构化表单的工具 + 流式 cursor。

### 14. Prompt Template：组件化管理 prompt
- **目标**：告别硬编码 prompt。
- **要点**：模板变量、few-shot、system/human/ai 消息组织。
- **产出**：把项目里的 prompt 全部模板化、可复用。

### 15. Runnable：把写逻辑变成组装 chain
- **目标**：理解 LangChain 的核心抽象 Runnable。
- **要点**：Runnable 接口、管道组合、并行/分支。
- **产出**：用 Runnable 重构前面的流程。

### 16. 实战练习 LCEL 组装 chain
- **目标**：熟练 LCEL（LangChain Expression Language）。
- **要点**：`|` 管道、RunnableMap/Branch、流式与批处理。
- **产出**：用 LCEL 重写 RAG 链路，代码更声明式。

### 17. LangChain 整体总结：Agent 第一阶段学习完成
- **目标**：复盘阶段一，串成知识网。
- **要点**：Tool / RAG / Memory / Prompt / Runnable 的关系图。
- **产出**：一份个人的「LangChain 速查表」。

# 阶段二：工程化与后端（第 18–26 节）
> 把玩具脚本变成能上线的服务，引入 Nest、流式接口、部署、全文检索。

### 18. Nest + LangChain 实现基于 SSE 的流式 AI 接口
- **目标**：用企业级 Node 框架包装 agent 能力。
- **要点**：Nest 模块/控制器/服务；SSE 流式返回 token；与前端对接。
- **产出**：一个 `/chat` 流式接口。

### 19–20. Nest + tool 实现「OpenClaw 同款」定时任务（上 / 下）
- **目标**：让 agent 具备定时 / 后台任务能力。
- **要点**：Nest 定时任务（cron）、任务队列、tool 与调度结合；持久化任务状态。
- **产出**：一个可配置的定时 agent 任务系统。

### 21. 给 Agent 加上语音交互：ASR + 流式 TTS
- **目标**：多模态交互入门。
- **要点**：ASR（语音转文字）、流式 TTS（文字转语音）、与流式对话拼接。
- **产出**：一个可语音对话的 agent。

### 22. AGUI 协议：Vercel AI SDK + LangChain 实现流式组件渲染
- **目标**：让前端「边生成边渲染组件」。
- **要点**：Vercel AI SDK 的 UI streaming、AG-UI 事件协议、generative UI。
- **产出**：前端流式渲染 agent 返回的结构化组件。

### 23. 图编排引擎：LangGraph 和多 Agent 架构
- **目标**：从「链」升级到「图」。
- **要点**：StateGraph、节点/边/条件边、循环与回退；多 agent 协作模式。
- **产出**：用 LangGraph 重写一个带分支决策的流程。

### 24. Agentic RAG：基于 LangGraph 的自主决策 RAG 闭环
- **目标**：让模型自己决定「要不要检索、检索够没够、要不要再查」。
- **要点**：检索质量评估、重写 query、自我反思循环。
- **产出**：一个会自主决策的 RAG agent。

### 25. 基于 Docker Compose 的本地开发提效和生产部署
- **目标**：一键起全套依赖、可部署。
- **要点**：compose 编排（应用 + Milvus + ES + DB）、环境区分、镜像构建。
- **产出**：`docker compose up` 拉起整套环境。

### 26. ElasticSearch 全文检索：倒排索引 + IK 分词 + BM25
- **目标**：补齐「关键词检索」这条腿。
- **要点**：倒排索引原理、IK 中文分词、BM25 打分；与向量检索的互补。
- **产出**：用 ES 实现全文检索接口。

# 阶段三：高级 RAG / 记忆 / 可观测（第 27–34 节）
> 把检索做强、把记忆做长、把系统做可观测，逼近「企业级」。

### 27. 混合检索 RAG：多路召回 + 重排模型
- **目标**：显著提升检索准确率。
- **要点**：向量检索 + BM25 多路召回、RRF 融合、rerank 重排模型。
- **产出**：一个混合检索 + 重排的 RAG。

### 28. Neo4j 知识图谱和 Graph RAG
- **目标**：用图结构表达实体关系，支持关系型问答。
- **要点**：Neo4j / Cypher、实体抽取建图、Graph RAG 检索。
- **产出**：基于知识图谱的问答。

### 29. LangSmith 全链路观测：从 Agent 调试到 RAG 量化评估
- **目标**：让 agent 可调试、可评估。
- **要点**：trace 全链路、数据集与评估指标、RAG 质量量化（召回/忠实度）。
- **产出**：接入 LangSmith，跑一次评估报告。

### 30. DeepAgents：开箱即用的 skill、上下文压缩等 middleware
- **目标**：用更高层框架快速搭复杂 agent。
- **要点**：skill 机制、上下文压缩、中间件思想。
- **产出**：用 DeepAgents 搭一个带 skill 的 agent。

### 31. DeepAgents 实战：多 Agent 架构的深度调研助手
- **目标**：综合运用多 agent 做「深度研究」。
- **要点**：规划 agent / 执行 agent / 汇总 agent 分工；长任务编排。
- **产出**：一个能自动调研并产出报告的助手。

### 32. PostgreSQL：AI 时代最适合的数据库
- **目标**：用一个库同时承载关系数据 + 向量（pgvector）。
- **要点**：pgvector 扩展、JSONB、与业务表统一存储。
- **产出**：用 Postgres 同时存业务数据和向量。

### 33. Redis：实现 Agent 短期记忆存储
- **目标**：高性能短期记忆 / 会话状态。
- **要点**：会话上下文缓存、TTL、消息历史结构。
- **产出**：把对话短期记忆迁到 Redis。

### 34. Mem0：分层记忆 + 三路召回的长期记忆方案
- **目标**：给 agent 装上「长期记忆」。
- **要点**：分层记忆（用户/会话/事实）、三路召回、记忆写入与遗忘。
- **产出**：集成 Mem0，实现「越用越懂你」的记忆。

# 综合实战：企业级知识库 Agent（贯穿全程）
把上面各模块拼成一个完整项目，建议作为结课大作业：
- **数据层**：loader/splitter 导入多源文档 → 向量化入库（Milvus / pgvector）+ ES 全文索引
- **检索层**：混合检索（向量 + BM25）+ rerank 重排（第 27 节）+ 可选 Graph RAG（第 28 节）
- **Agent 层**：LangGraph 编排 Agentic RAG（第 23–24 节），DeepAgents 做多 agent 调研（第 30–31 节）
- **记忆层**：Redis 短期（第 33）+ Mem0 长期（第 34）
- **服务层**：Nest 提供 SSE 流式接口（第 18、35），前端用 Vercel AI SDK 做 generative UI（第 22）
- **可观测**：LangSmith 全链路 trace + RAG 评估（第 29）
- **交付**：Docker Compose 一键部署（第 25）

# 建议学习节奏
- 阶段一：2–3 周（每天 1 节，重在敲代码）
- 阶段二：2 周（开始关注工程化）
- 阶段三：2–3 周（难度上升，配合项目）
- 阶段四 + 综合实战：2 周
- 总计约 8–10 周，产出一个可放进简历的企业级知识库 Agent 项目。
